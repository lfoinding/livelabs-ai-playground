# LiveLabs Screenshot Workflow Guide

Estimated Time: 10 minutes

## Introduction

Use this guide when you want an overview of the screenshot workflow for LiveLabs authoring.

It points you to the full workshop-style lab and summarizes the core capture pattern so you can find the right starting place quickly.

### Objectives

- understand what the screenshot workflow helps authors do
- find the full screenshot lab in this repo
- know the core capture pattern before opening the detailed lab

## Task 1: Start With The Right Lab

1. Use the full screenshot lab here:

    - `skills/how-to/screenshot-skill/livelabs-screenshots-with-codex/livelabs-screenshots-with-codex.md`

2. Use that lab when you want the full step-by-step workflow for:

    - capturing workshop-ready screenshots
    - following the learner flow in a real browser
    - adding annotations such as red boxes
    - blurring sensitive information
    - saving stable image assets for workshop markdown

## Task 2: Remember The Core Pattern

1. Start from the workshop markdown or the exact LiveLabs URL.

2. Give the agent the lab or task range to follow.

3. Tell it which states need screenshots.

4. Name any image edits you need, such as:

    - red boxes
    - arrows
    - blur rules
    - highlight callouts

5. Save the screenshots under a stable output folder and keep a manifest of what was captured.
